## Send an automatic message at the end of a session

The automatic message is sent as soon as the session ends. 
From this message, the participants can find a link to the end of session page that sums-up all the session information and shared files.

1. In the left-hand menu, click the service you want.
2. Click on the **Configuration** tab.
3. In the menu **Ending a session**, tick the box **Send an automatic message at the end of the session**.
4. Click **Save**. 
 
 ![](../../../.gitbook/assets/EN-portal-send-automatic-message-end-session.png)

{% hint style="success" %}
Now, the guests of each session will receivea message when the session ends.
{% endhint %}

Of course, this automatic message is completely customizable from the **Notification templates** tab. You can choose whether you want to add more information or change the style.

##  Display the shared files on the end of session page

The end of session page can be useful as it displays a sum-up (name of the organizer, day and time of the meeting...) of the passed session. 
In addition to that piece of information, you can configure to display the files shared during the session. 
This is really helpful for the participants as they can download the files here if they forgot to do it during the session.

1. In the left-hand menu, click the service for which you want to configure the end of session page.
2. Click on the **Configuration** tab.
3. In the menu **Files to download after the session**, choose the type of files you want to display on the end of session page.
4. In the menu **Ending a session**, tick the box **Display the shared files on the end of session page**. 

    | ![](../../../.gitbook/assets/tip.png) | For even more security, you can protect the access to the end of session page with a **security code**:
 
Tick the box **Access to the end of session page with a security code**.
 

    | ![](../../../.gitbook/assets/ok.png) | The participant will receive a message with the code and will be asked to enter it before accessing the end of session page. |
    | --- | --- | |
    | --- | --- |
5. Click **Save**.

![](../../../.gitbook/assets/EN-portal-display-shared-files-end-session-page.png)

{% hint style="info" %}
This option often goes with **[Send an automatic message at the end of the session](configure-the-end-of-teleconsultation.md)**, but not necessarily. 
 
For example, in this specific case: 
After receiving the **automatic message** and clicking the link to access the **end of session page**, the participant is asked to enter a security code sent in another message.
 
After **Confirming**, the participant accesses the end of session page.
{% endhint %}

## Allow to share again the files after a session 

1. In the left-hand menu, click the service you want.
2. Click on the **Configuration** tab.
3. In the menu **Ending a session**, tick the box **Send a download link with the shared files [from the portal]**. 

    | ![](../../../.gitbook/assets/tip.png) | For more security, you can protect the download link with a security code:
 
Tick the box **Access to the download page with a security code**.
 

    | ![](../../../.gitbook/assets/ok.png) | The participant will receive a message with the code and will be asked to enter it before accessing the download page. |
    | --- | --- | |
    | --- | --- |
4. Click **Save**. 
 
 ![](../../../.gitbook/assets/EN-portail-share-files-again-through-session-page.png)

{% hint style="success" %}
From the teleconsultation page on the portal, the organizer will be able to send a download link to the participants.
{% endhint %}
