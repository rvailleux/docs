# Glossary

| Term | Definition |
| :--- | :--- |
| **Agent** | A user who receives and handles video assistance requests from guests or customers. |
| **Back-Office** | The IT operator administration interface for managing tenants, global configuration, and platform-level settings. Distinct from the per-tenant Apizee portal. |
| **Common invitation** | An invitation sent with a customizable form (guest name, contact details, service selection) — used when more context is needed before the session starts. Compare with Quick invitation. |
| **Conference** | A real-time video session with one or more participants. |
| **DIAG / Diag Help Desk** | Apizee's video assistance product for customer support and help desk operations. |
| **Embed** | The Apizee product for embedding video sessions into third-party web applications using the Embed SDK and portal. Replaces the standalone Apizee widget for custom integrations. |
| **Flow / Workflow** | A configurable sequence of steps that governs how a session is initiated, routed, and concluded in the Embed portal. |
| **Guest** | A person who joins a video session without an Apizee account, typically via an invitation link. |
| **IT Operator** | An administrator with access to the Back-Office who manages tenants and platform configuration across multiple organizations. |
| **Organizer** | The user who creates and manages a multi-participant session. |
| **Quick invitation** | A one-click invitation sent by email and/or SMS with minimal form fields — used for fast, low-friction session starts. Compare with Common invitation. |
| **Requester** | The person initiating an assistance request (often the guest). |
| **Service** | A configured queue or line of business to which tickets and invitations are assigned. |
| **Service Account** | A non-personal technical account used by integrations or automated systems to authenticate with the Apizee API on behalf of a tenant. |
| **Session** | A single video assistance interaction between participants. |
| **Supervisor** | A user who oversees agent activity, reviews tickets, and manages team availability. |
| **Tenant** | An isolated Apizee environment for a single organization, managed within the Back-Office. Each tenant has its own users, services, and configuration. |
| **Timeline** | A chronological record of events and actions that occurred during a session. |
| **Ticket** | A record created for each assistance request, containing guest information, session history, and shared files. |
| **Waiting room** | A holding area where guests wait before being admitted to a session by the agent or organizer. |
| **WebRTC** | Web Real-Time Communication — the browser protocol used by Apizee to enable peer-to-peer audio, video, and data exchange without plugins or app installs. |
| **Whiteboard** | A collaborative drawing surface available during a session for annotations and highlighting. |
